# Accessibility Rubric

## Overview
This rubric evaluates how well educational content meets accessibility standards and serves diverse learners, including those with disabilities or different learning needs.

## Scoring Scale
- **4 - Exemplary**: Exceeds accessibility standards, fully inclusive
- **3 - Proficient**: Meets accessibility standards with minor gaps
- **2 - Developing**: Some accessibility features but significant gaps
- **1 - Inadequate**: Poor accessibility, excludes many learners

## Evaluation Criteria

### 1. Visual Accessibility (25%)

#### Exemplary (4)
- Perfect color contrast ratios (WCAG AAA)
- All images have detailed alt text
- Text scalable without loss of functionality
- Multiple visual representations provided

#### Proficient (3)
- Good color contrast (WCAG AA)
- Most images have appropriate alt text
- Generally scalable text
- Some alternative visual options

#### Developing (2)
- Some contrast issues
- Basic alt text present
- Limited scalability
- Few visual alternatives

#### Inadequate (1)
- Poor color contrast
- Missing or inadequate alt text
- Fixed text sizes
- No visual alternatives

### 2. Auditory Accessibility (25%)

#### Exemplary (4)
- All audio/video fully captioned and transcribed
- Visual indicators for audio cues
- Multiple audio quality options
- Sign language interpretation available

#### Proficient (3)
- Most media captioned/transcribed
- Some visual indicators
- Good audio quality
- Generally accessible audio

#### Developing (2)
- Partial captions/transcripts
- Limited visual cues
- Variable audio quality
- Some accessibility gaps

#### Inadequate (1)
- No captions or transcripts
- No visual alternatives
- Poor audio quality
- Audio-dependent content only

### 3. Navigation and Structure (25%)

#### Exemplary (4)
- Perfect keyboard navigation throughout
- Clear, consistent heading hierarchy
- Skip links and landmarks provided
- Multiple navigation methods available

#### Proficient (3)
- Good keyboard navigation
- Generally clear structure
- Most navigation aids present
- Alternative navigation options

#### Developing (2)
- Some keyboard access issues
- Inconsistent structure
- Basic navigation aids
- Limited navigation options

#### Inadequate (1)
- Poor keyboard support
- Confusing structure
- Missing navigation aids
- Single navigation method only

### 4. Cognitive Accessibility (25%)

#### Exemplary (4)
- Clear, simple language options
- Consistent, predictable interface
- Multiple complexity levels available
- Comprehensive support tools provided

#### Proficient (3)
- Generally clear language
- Mostly consistent interface
- Some complexity options
- Good support tools

#### Developing (2)
- Sometimes complex language
- Some interface inconsistencies
- Limited complexity options
- Basic support tools

#### Inadequate (1)
- Unnecessarily complex language
- Confusing interface
- Single complexity level
- Minimal support tools

## Implementation Guidelines

### Assessment Process
1. Run automated accessibility checkers
2. Test with screen readers
3. Verify keyboard navigation
4. Check cognitive load factors
5. Calculate weighted score

### Common Issues to Address
- **Color-only information**: Using color as sole indicator
- **Missing alt text**: Images without descriptions
- **Keyboard traps**: Areas where keyboard users get stuck
- **Time limits**: Fixed time constraints without extensions

### Improvement Recommendations
- Follow WCAG 2.1 guidelines (Level AA minimum)
- Provide multiple content formats
- Use clear, consistent layouts
- Test with actual assistive technologies

## Accessibility Standards Checklist

### WCAG Principles

#### Perceivable
- [ ] Text alternatives for non-text content
- [ ] Captions for videos
- [ ] Audio descriptions available
- [ ] Sufficient color contrast
- [ ] Resizable text

#### Operable
- [ ] Keyboard accessible
- [ ] No seizure-inducing content
- [ ] Sufficient time limits
- [ ] Clear navigation
- [ ] Descriptive headings

#### Understandable
- [ ] Readable text
- [ ] Predictable functionality
- [ ] Input assistance
- [ ] Error identification
- [ ] Clear instructions

#### Robust
- [ ] Valid markup
- [ ] Compatible with assistive technology
- [ ] Progressive enhancement
- [ ] Fallback options
- [ ] Future-proof coding

## Assistive Technology Support

### Screen Reader Compatibility
- [ ] Proper heading structure
- [ ] Meaningful link text
- [ ] Form labels and instructions
- [ ] Table headers defined
- [ ] ARIA labels where needed

### Keyboard Navigation
- [ ] Tab order logical
- [ ] Focus indicators visible
- [ ] Skip links provided
- [ ] No keyboard traps
- [ ] Shortcuts documented

### Visual Adaptations
- [ ] High contrast mode support
- [ ] Dark mode option
- [ ] Zoom to 200% without horizontal scroll
- [ ] Focus indicators visible
- [ ] Color not sole indicator

### Cognitive Support
- [ ] Consistent navigation
- [ ] Clear error messages
- [ ] Progress indicators
- [ ] Help readily available
- [ ] Simple language option

## Compliance Levels

### Minimum (Level A)
- Basic accessibility features
- Major barriers removed
- Essential functionality accessible

### Recommended (Level AA)
- Good accessibility coverage
- Most users accommodated
- Industry standard compliance

### Enhanced (Level AAA)
- Exceptional accessibility
- Maximum inclusivity
- Above standard compliance
